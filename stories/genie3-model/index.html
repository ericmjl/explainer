main scaffold

Residues become a flat coordinate-token axis before the trunk sees them.

Non-atomized residues contribute one CA token. Atomized known residues contribute CA plus sidechain heavy atom tokens, each with its own coordinate, frame, single slot, and pair-representation row/column.

frame semantics

Rotations are internal; translations are the diffusion output.

The structure stack predicts quaternion and translation updates for every token frame. Those rotations affect later IPA layers inside the denoiser call, but the sampler receives only final frame translations.

tokenization

Atomized residues are sparse

The feature builder inserts CA, then appends `atom_type[4:]` and `gt_atom_positions[4:]` for sidechain heavy atom tokens.

pair axis

Sidechain atoms own pair slots

Pair features are built as `N_token x N_token`; sidechain atom tokens are not hidden behind a residue-level sidechain decoder.

structure stack

IPA updates token frames

Each structure layer runs IPA on current token frames, then composes a learned quaternion and translation update into the frame state.

The denoiser returns translations

After the structure stack, Genie3 returns `fi.trans`. The next diffusion step recomputes frames from Cartesian coordinates.

Sidechain Scope

Genie3 generates sidechain coordinates when the residue identity is known and the residue is atomized. Unknown-sequence binder design avoids this by staying CA-token level for generated binder residues.
